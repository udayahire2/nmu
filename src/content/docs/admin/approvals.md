## Approval Workflow

Faculty-uploaded resources enter a **pending** state and are not visible to students until an admin approves them.

1. Faculty submits a resource via `/dashboard/faculty`
2. Admin receives a notification badge on the dashboard
3. Admin navigates to `/admin/approvals`
4. Admin previews the resource and reviews its metadata
5. Admin clicks **Approve** or **Reject**
6. Approved resources become visible to all students immediately

## Reviewing a Submission

Each submission card shows:

- **Title** and **description**
- **Type** (PDF / Notes / Video / Link)
- **Subject** and **branch/semester** it belongs to
- **Uploaded by** (faculty name)
- **Uploaded at** timestamp

Click **Preview** to open the resource in a viewer before making a decision.

## Rejection Notes

When rejecting, you can optionally provide a reason. The faculty member will see this note on their dashboard so they can revise and resubmit.

## Bulk Actions

Select multiple submissions and use **Approve All** or **Reject All** to process them in bulk from the approvals list view.
