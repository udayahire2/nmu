### Procedural vs Object-Oriented Programming

Procedural Programming (PP) and Object-Oriented Programming (OOP) are two main ways to write code. PP treats the program as a list of steps and functions. OOP treats it as objects that hold data and actions together. In C++, you can use both, but OOP is better for larger projects.

### Procedural Programming (PP)
- Focuses on "what to do" – a sequence of steps.
- Data and functions are separate.
- Example: Simple C program with functions like `calculateTotal()`.
- Best for small, easy tasks like math calculations.

### Object-Oriented Programming (OOP)
- Focuses on "real things" – objects with data and functions inside.
- Objects model everyday items, like a "Student" with name and grades.
- Example: C++ class `Student` with functions like `showGrades()`.
- Best for big, complex programs like games or apps.

### Key Differences (Simple Explanation)
1. **Style**: PP is step-by-step like a recipe. OOP is like building blocks.
2. **Data Safety**: In PP, data can be changed anywhere. In OOP, data is protected inside objects.
3. **Reuse**: PP reuses functions only. OOP reuses whole classes easily.
4. **Size**: PP works for tiny programs. OOP handles large teams and big apps.
5. **Changes**: PP is hard to update later. OOP allows easy additions without breaking old code.
