### Class

A **class** is a user-defined data type that acts as a **blueprint** for creating objects. It groups **data members (variables)** and **member functions (methods)** into a single unit. A class itself does not occupy memory until an object is created.

### **Object**

An **object** is an **instance of a class**. It represents a real-world entity and occupies memory. Objects are used to access the data members and member functions defined in a class.

---

### **Key Points**

- Class defines **structure and behavior**.
    
- Object represents **actual usage** of that structure.
    
- Multiple objects can be created from the same class.
    
- Each object has its **own copy of data members**.
    

---

### **Example (Conceptual)**

- **Class:** `Student`
    
- **Objects:** `s1`, `s2`, `s3`
    

Here, `Student` defines what a student is, while `s1`, `s2`, and `s3` represent individual students.

---

### **Difference Between Class and Object**

|**Class**|**Object**|
|---|---|
|Blueprint or template|Instance of a class|
|Does not occupy memory|Occupies memory|
|Logical entity|Physical entity|
|Declared once|Can be created many times|
