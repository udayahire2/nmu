### Introduction to Object-Oriented Programming (OOP)

Object-Oriented Programming (OOP) is a method of writing code that focuses on using "objects" to represent real-world items. In C++, OOP helps structure programs in a way that is easy to build, maintain, and expand.

### Benefits of OOP

- **Code Reuse**: Allows you to use the same code multiple times without repeating it.
- **Clear Structure**: Organizes code like everyday objects, making it simpler to read and update.
- **Manages Complexity**: Breaks large problems into smaller, connected parts for better handling.
- **Efficient Changes**: Modifies one section without affecting the entire program.

### Core Concepts in OOP

1. **Class**: A template that defines what an object can have (data) and do (actions). For example, a "Car" class might include color (data) and drive (action).

2. **Object**: An actual instance created from a class. Using the "Car" class, you can make objects like "RedCar" or "BlueCar," each with unique details.

3. **Encapsulation**: Keeps an object's internal workings hidden. You use the object without needing to know its inner mechanics, like operating a phone without understanding its circuits.

4. **Inheritance**: Enables a new class to take features from an existing one. A "Truck" class can inherit from "Vehicle" and add its own traits, like cargo space.

5. **Polymorphism**: Allows the same operation to behave differently for various objects. For instance, a "draw" function might create a circle for one shape and a square for another.

### OOP in C++ Practice

In C++, you create classes with variables (for data) and functions (for actions). Start with basic examples to see how objects interact. This approach makes coding more intuitive and practical.
